## Goal
Implement real, persistent "My list" (Want / Have / Gift) and wire /family to show real counts and items; wire Discover "Save idea" and "Have them" to persist to `user_list_items`.

## Ground truth
- **Save idea / Have them CTAs:** Rendered in `web/src/components/discover/CategoryCarousel.tsx` ("Save idea", "Have them"). Handled in `web/src/app/discover/[months]/DiscoveryPageClient.tsx`: `handleSaveCategory`, `handleHaveThemCategory`, `handleSaveToList`, `handleHaveItAlready`; replay in `runReplayForAction`.
- **Existing save tables:** `public.user_saved_products` and `public.user_saved_ideas` in `supabase/sql/202602190000_subnav_saves_and_consent.sql`, actively used. This PR adds a single canonical table `user_list_items` with Want/Have/Gift semantics, backfills from those tables, and switches all writes to it; `get_my_subnav_stats` now counts from `user_list_items`.

## UI semantics (implemented)
- **Want** toggles `want` true/false.
- **Have** toggles `have` true/false (Discover "Have them" sets have=true).
- **Gift** is a flag within Want: toggling Gift on auto-sets Want=true; toggling Want off clears Gift (gift implies want).

## Rollback
- **Code:** Revert this PR.
- **DB:** Only if safe and you need to remove the table:  
  `DROP TABLE IF EXISTS public.user_list_items CASCADE;`  
  Then re-run or restore `get_my_subnav_stats` from `202602190000_subnav_saves_and_consent.sql` if you want it to read from `user_saved_products` / `user_saved_ideas` again.
