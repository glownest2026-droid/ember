## Goal
Establish Ember's staging frontend/deployment foundation by confirming current truth, locking the staging topology, and preparing/executing the minimum repo + branch + Vercel pathway for a long-lived staging lane. No staging backend or seed data in this PR.

## What changed
- **docs/staging/current-truth.md** — One-page snapshot: repo root, app root (web/), build commands, production branch (main), env vars and Supabase wiring paths.
- **docs/staging/topology-decision.md** — Locked decision: main = production; staging branch = stable staging URL; feature branches/PRs = preview. Branch-based staging; no new providers.
- **docs/staging/runbook.md** — Founder-safe clerical steps: create/verify staging branch, Vercel Root Directory and Staging environment, env vars, promote-to-staging flow, redeploy note.
- **docs/staging/environment-matrix.md** — All env keys referenced in code (no fabricated names); use when setting Vercel Staging env.
- **PROGRESS.md** and **state/latest.json** — Updated with this PR receipt, QA steps, rollback.

Also: **staging** branch created and pushed from main (long-lived branch for Vercel to assign stable staging URL).

## Out of scope (follow-on PRs)
- Staging Supabase project/connection
- Seed or test data for staging
- Feature flags or environment badge in app
- CI/CD redesign

## Verification
1. Build: `cd web && pnpm install && pnpm run build` passes.
2. Open Vercel preview URL; app loads, basic navigation works.
3. Review docs/staging/* in GitHub.
4. After merge: follow docs/staging/runbook.md for Vercel Staging env and env vars.

## Rollback
Revert this PR. Delete docs/staging/ if desired. No production or DB changes.
